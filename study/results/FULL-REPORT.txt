SZZ blame links vs JIT features:

  SZZ traces fix commits back to the commits that introduced the
  bug via git blame. It only produces rows for PRs that introduced
  a bug that was LATER FIXED. The ~65K SZZ rows are blame links
  (fix → bug-introducing commit), not PR counts. Many PRs never
  introduced a traced bug, so they have no SZZ rows.

  JIT features (lines added, entropy, developer experience, etc.)
  are computed from diff metadata and can be calculated for ANY PR.
  The ~109K JIT rows cover nearly all PRs in the dataset.

  16 of 119 repos have zero SZZ coverage because their squash-merge
  workflows cause GitHub's merge SHAs to be garbage-collected,
  making them unreachable via git blame.
